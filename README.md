# しりとり

ブラウザで遊べるシンプルなしりとりアプリです。
**jig.jp 2026 サマーインターン選考課題** として制作しました。

🔗 **公開URL: https://shitake-zense.github.io/shiritori-app/**

ビルド・依存関係なしの Vanilla JS（ES モジュール）による静的サイトで、GitHub Pages にデプロイしています。

---

## 遊び方

1. ランダムな初期単語が表示されます。
2. その単語の**最後の文字**から始まるひらがな単語を入力して送信します。
3. 正しく続く限りゲームは継続。次の単語を入力していきます。
4. 次のいずれかで**あなたの負け**になります。
   - 「ん」で終わる単語を入力した
   - すでに使った単語をもう一度入力した（重複）
5. 「はじめから」でいつでも新しいゲームを開始できます。

### 遊び方モード（3種）

「遊び方」セレクトで接続ルールを切り替えられます（選んだモードは画面最上部の見出しにも反映されます）。

| モード | 接続ルール | 例 |
|---|---|---|
| **しりとり**（標準） | 前の単語の**末尾**の文字から始まる単語 | りんご → ごりら → らっぱ |
| **あたまとり** | 前の単語の**先頭**の文字で**終わる**単語 | りんご → はかり → かれは → わか |
| **しりとりすぎ** | 前の単語の末尾と次の単語の先頭の**重なり**でつなぐ。重なった文字数が**得点**になり、最大ターンまでに高得点を競う | りんご → ごりら → りらっくす → くすり（重なり1, 2, 2…で加点） |

いずれのモードも「ん」終了・重複は負け。**文字数しばり・解答時間制限・単語チェック**と併用できます。しりとりすぎのソロは**スコアアタック**（最大ターンまで生存で最終スコア確定）、オンラインは**ポイントで勝敗**（同点は引き分け）です。

## 機能

| 機能 | 説明 |
|---|---|
| 文字一致判定 | 前の単語の末尾文字から始まるかを判定。末尾「ー」・小書き文字（ぁゃっ等）も正規化して接続 |
| 「ん」終了・重複終了 | しりとりの基本ルールで勝敗を判定 |
| ひらがなバリデーション | ひらがな（長音符含む）2文字以上のみ受付 |
| ランダム初期単語 | ゲーム開始時に初期単語を自動選択 |
| 対戦中の単語チェーン表示 | その対戦でつないだ単語を一覧表示 |
| 連続成功カウント | 何回続けてつなげているかを盤面に表示。終了時はリザルト文言に切替 |
| 履歴の保存・一覧 | 過去のゲーム結果を localStorage に保存（最大20件・新しい順）。**そのときのルール設定も併記**（しりとりすぎは得点・勝敗バッジも）。クリア可能 |
| 単語チェックモード | ON にすると約45,000語の辞書に存在する実在単語のみ許可（IPAdic 由来） |
| **遊び方モード** | 独自ルール。**あたまとり**（接続反転）・**しりとりすぎ**（重なり加点＋最大ターンで点数勝負）を選択可能。見出しもモードに連動 |
| **文字数しばり** | 独自ルール。「ちょうど3文字」または「N文字以上（3〜5）」を選択可能 |
| **解答時間制限** | ON/OFF切替（既定OFF）。ONで解答時間を入力（3〜100秒）。時間切れでその手番のプレイヤーが負け |
| 開始ボタン・ルール固定 | ルールを設定して「しりとりを始める」で開始。開始後はルール変更不可 |
| **オンライン対戦** | ルームコード方式の2人対戦（Firebase Realtime Database）。手番・勝敗・得点・時間制限をリアルタイム同期 |

## 技術構成

- **Vanilla JS（ES モジュール）** + プレーン CSS。ビルド・パッケージマネージャ・依存関係なし。
- オンライン対戦のみ **Firebase Realtime Database** を CDN から ES モジュールで読み込み。
- **GitHub Pages** で配信。和モダン（墨×朱×和紙）のデザイン（Webフォント: Shippori Mincho B1 / Zen Kaku Gothic New）。
- ロジックと DOM を意図的に分離。この分離のおかげでオンライン対戦（online.js）を後付けで重ねられました。

```
index.html          UI（ソロ設定/盤面・オンライン作成/対戦を画面切替）
css/style.css        和モダンのスタイル
js/game.js           DOM非依存の純粋ロジック（文字正規化・判定 judge()）
js/main.js           DOM制御とゲーム進行（ソロ/オンライン2モード・画面制御 applyView）
js/online.js         オンライン対戦ロジック（Firebase RTDB・ルーム同期・権威判定）
js/firebase-config.js  Firebase設定（databaseURL 必須）
js/history.js        履歴の localStorage 永続化（ルール設定も保存）
js/dictionary.js     初期単語・辞書ロジック（words.json を非同期ロード）
words.json           単語チェック用辞書（約45,000語）
tools/build-dictionary.mjs  IPAdic から words.json を生成するスクリプト
docs/FIREBASE.md     オンライン対戦のFirebase設定手順
```

判定の中心は `game.js` の `judge()` で、戻り値 `{ ok, reason?, end?, points? }` を軸に状態遷移します（`end: "lose"` =「ん」終了・重複による敗北。文字数しばり等の違反は `ok: false` の再入力要求として返す。`points` はしりとりすぎの重なり得点）。接続方向は `opts.mode`（標準／あたまとり／しりとりすぎ）で切り替わります。オンラインでも同じ `judge()` を transaction 内で権威評価するため、ルールは1か所（game.js）に集約されます。

## ローカルでの実行

`index.html` は `type="module"` を使うため、`file://` 直開きでは動きません。ローカル HTTP サーバ経由で開いてください。

- VSCode の **Live Server** 拡張で `index.html` を開く、もしくは
- 任意の静的サーバ（例: `python -m http.server`）を起動してアクセス

```bash
python -m http.server 8000
# → http://localhost:8000/ を開く
```

## オンライン対戦

ルームコード方式の2人対戦です。片方が「ルームを作る」でコードを発行し、もう片方がそのコードで参加します。手番・勝敗・得点・解答時間はリアルタイムに同期されます。

**公開URLではそのまま2人対戦できます**（Firebase 設定済み・コミット済みのため、遊ぶ側のセットアップは不要。ルームコードを共有するだけ）。

> Firebase の設定（[`docs/FIREBASE.md`](docs/FIREBASE.md)）が要るのは、このリポジトリを fork して**自分の Firebase プロジェクトで自前ホストする場合**だけです。その場合 `js/firebase-config.js` を未設定にすると、オンライン対戦は無効になり**ひとりプレイのみ**動作します。

## デプロイ

`main` ブランチへの push が GitHub Pages（Source: `main` / root）に自動反映されます。

## 辞書の更新

単語チェック用辞書 `words.json` は IPAdic（mecab-ipadic）から生成しています。語彙を更新する場合:

```bash
node tools/build-dictionary.mjs
```
